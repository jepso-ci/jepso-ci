Jepso CI

Jepso CI is a simple tool to perform cross unit testing. It's free for open source libraries and tests on over 60 browser/os combinations.

It can provide you with a compatibility badge in a range of formats. Such as this one (for native JSON support).

Integrating your test framework

It is simple to integrate any JavaScript testing framework. All that's required is that you set the value window.testsPassed to either true or false to indicate the success or failure of your tests.

If you have lots of asynchronous tests, you should also set the value window.completedTests = 0 and then increment it (window.completedTests++;) after each test, otherwise we'll assume the tests have hung and timeout.

Enabling CI

To enable CI your project must first be publicly hosted on GitHub. Next you must add a file in the root of the repository called .jepso-ci.json and it should look something like the following:

{
  "url": "/tests/test.html"
}

The url must begin with a / and be a relative path name to an html file

To have tests run each time you push changes you'll need to add a webhook in the admin for your github repository. Pointing to https://jepso-ci.com/api/hook
